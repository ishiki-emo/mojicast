# コラボ字幕 設計メモ（ROADMAP #4 案B）

各参加者が自分のPCで Mojicast を起動し、**確定字幕テキストだけ**をリレー経由で
共有して、全員分の字幕を各自のOBSに別々のスタイルで表示する。

音声は一切送らない。各自のクリーンなマイク音声をローカル認識するため、
精度は最高・話者の帰属は完璧・負荷は各PCに分散・帯域はテキストのみでほぼゼロ。

---

## 1. アーキテクチャ

```
[えも PC]                                     [ゲスト PC]
  マイク → Mojicast（ローカル認識）             マイク → Mojicast（ローカル認識）
     │ 確定字幕を送信                              │ 確定字幕を送信
     ▼                                           ▼
  ┌────────────── リレー（Cloudflare Workers + Durable Objects）─────────────┐
  │  部屋コードごとに1つの「部屋」。受け取ったメッセージを部屋の全員へ配るだけ  │
  └──────────────────────────┬───────────────────────────────────────────┘
     │ 全員分の字幕(WebSocket)                     │ 全員分の字幕
     ▼                                           ▼
  OBS オーバーレイ（話者ごとにスタイル/ボックス）   OBS オーバーレイ（同じく）
```

### リレーの技術詳細
- **Cloudflare Worker**: WebSocketへのアップグレードを受け、部屋コードから
  対応する Durable Object へルーティングするだけの薄い層
- **Durable Object（部屋）**: 接続中クライアントの一覧を持ち、受信メッセージを
  全員へブロードキャストする。数十〜百行規模
  - WebSocket Hibernation API を使い、待機中の課金・リソースを最小化
  - 字幕は**保存しない**（素通しのみ・ログ無効）。部屋は全員切断で消滅
- **メッセージ形式**（JSON・1行1メッセージ）:
  ```json
  { "type": "final", "speaker": "えも", "fid": 123,
    "text": "こんばんは", "ts": 1234567890 }
  { "type": "translation", "speaker": "えも", "fid": 123, "text": "Good evening" }
  { "type": "presence", "members": ["えも", "ゲストA"] }
  ```
  - 英訳は各自のPCが自分の発話分を翻訳して後追い送信（負荷分散・fidで対応付け）
- **部屋コード**: 推測困難なランダム8文字程度をホスト側で生成。TLS必須（wss://）
- **コスト**: テキストのみ・数人・配信時間分の接続なので Cloudflare 無料枠で十分
- 遅延はリレー数十ms程度で、認識処理（数百ms）に比べ無視できる

### 代替案（リレー無し・最初の一歩に）
Cloudflare Tunnel / Tailscale でホストのMojicast APIへゲストが直接POSTする構成なら
サーバーデプロイ不要。ただし「全員のOBSに全員分を表示」にはリレー型が必要なので、
本命はリレー型（この代替はホストのOBSにだけ出す1画面コラボ用）。

## 2. Mojicast側の改修（実装タスク）

| # | 内容 | 規模 |
|---|---|---|
| 1 | リレーWorkerの実装・デプロイ（wrangler） | 小（数十〜百行） |
| 2 | 送信: 確定字幕/英訳をWSでリレーへ（コラボON時のみ） | 小 |
| 3 | 受信: リレーからのメッセージを既存SSEパイプラインへ合流 | 中 |
| 4 | overlay: 話者→プリセット/ボックスの割当て表示（resolve_styleの拡張） | 中 |
| 5 | コックピットUI: コラボグループ（表示名・部屋の作成/参加・メンバー表示） | 中 |
| 6 | 再接続・エラー処理（リレー断でもローカル字幕は止めない） | 小 |

合計 2〜3日級。v1は**確定字幕のみ**共有（認識中の薄文字は将来拡張。
帯域的には可能だが、まず表示がシンプルな確定のみで様子見）。

## 3. セキュリティ / プライバシー

- 字幕テキストがCloudflareを経由する（TLS・保存なし）。この点は使う人に明示する
- 部屋コードを知っている人しか入れない（コードは毎回使い捨て）
- 受信テキストはoverlayで textContent 描画（既存実装のままHTML注入不可）
- 表示名も無害化してから表示

## 4. ユーザーが準備する手順

### 開発者（えもさん）の一度きりの準備
1. Cloudflareアカウント作成（無料。メールだけでOK）
2. `wrangler`（Cloudflare CLI）でリレーWorkerをデプロイ
   （コードはリポジトリに同梱予定。コマンド2つ程度）
3. 発行されたリレーURL（`wss://…workers.dev/…`）をMojicastの既定値に設定
→ 以降、テスターやコラボ相手は**このリレーをそのまま使える**（各自のデプロイ不要）

### ホスト（部屋を作る人）
1. Mojicast を起動 → コックピットの「コラボ」で表示名を入れて **部屋を作成**
2. 発行された**部屋コードをDiscord等で相手に共有**
3. いつも通り ▶開始。OBSの設定は変更不要（自分のoverlayに全員分が出る）

### ゲスト（参加する人）
1. 各自のPCに Mojicast をインストール（無料・通常のセットアップと同じ）
   - PCがMojicastの動作要件を満たしていること（コラボによる追加負荷はほぼ無し）
2. 「コラボ」で表示名＋**部屋コードを入力して参加**
3. 自分のマイクを選んで ▶開始
4. 自分も配信するなら、自分のOBSにいつも通りoverlayを設定（全員分が出る）

### 前提・制約
- 参加者全員がWindows PC＋マイク＋Mojicast動作要件を満たすこと
- 初回はモデルDL（約1.2GB）が各自必要
- スマホ勢・Mojicastを入れられない相手は対象外（その場合は案A=ループバックで
  「相手まとめて1ボックス」が代替）

## 5. 決めごと（実装前に確定したい）
