## 1. 単語セットのプロファイル化 ＋ data/ フォルダ再編 ✅実装済み（2026-07-12・未リリース）
**規模: 中（≈1日） / v0.3.0 の幹候補**

配信の性質で単語セットを切り替える（雑談用 / ゲーム実況用 / 歌枠用 …）。

- 設計: 「共通」＋ `profiles/<名前>/` の**合成**（上書きでなく加算・衝突はプロファイル優先）
  - 名前・チャンネル名は「共通」に置けば全プロファイルで有効（重複登録の回避）
- コックピットにセレクタ（`config.word_profile`）、単語スタジオに共通/プロファイル切替
- **同時に実施**: ユーザーデータを `data/` フォルダへ集約
  - `config.json`・単語系・presets/boxes → `data/`、profiles/ も `data/` 配下
  - ルートが整理され「バックアップ = dataフォルダのコピー」になる
  - **初回起動時にルートの旧ファイルを data/ へ自動移行**（既存ユーザー対策・必須）
  - `logs/` と `models/` は現行のまま
- プロファイルがそのまま mojipack（#2）の共有単位になる → **実装順は プロファイル → mojipack**

## 4. 別ソフト「話題レーダー」（仮称）
**規模: PoC半日 / 本体2〜3日 / パッケージ肥大回避のため別アプリ**

小型ローカルLLMで「今の話題」を単語抽出し、OBSにふわっと浮かべるコンパニオンアプリ。

- Mojicast側は**改修ゼロ**: SSE `http://localhost:8765/events` で確定字幕を購読できる
- 構成: 直近3〜5分のローリングウィンドウ → 30〜60秒毎にLLMで話題語3つ →
  自前ポートでOBS用オーバーレイ配信
- ランタイム: llama.cpp / GGUF（torch不要・数十MB）＋ Q4量子化モデル（初回DL方式）
- **モデル本命: Qwen3-1.7B**（CPU実試験で実用性確認済み・2026-07-12）
- CPU/GPU切替: `n_gpu_layers`（0=CPU / -1=全GPU）。バックエンドは **Vulkan推奨**
  （NVIDIA/AMD/Intel対応・同梱サイズ小。CUDAは数百MB膨らむ）
- **既定はCPU**（ゲーム配信中のGPUはゲーム+NVENCで激務）。
  「GPUを使う」トグル既定OFF＋GPU初期化失敗時はCPUフォールバック。
  雑談配信・2PC構成ではGPUが快適
- CPU配慮: 低スレッド・低優先度のバースト処理でASRを邪魔しない
- 発展: 話題語を `/api/effects` に流して字幕内を自動ハイライト（API既存）

## 6. コラボ字幕（複数話者の同時・別々字幕）
**規模: 案A=中 / 案B=中〜大 / 遠い将来枠・テスターの需要を見て**

Discordコラボで自分＋ゲストの字幕を別々のボックスに出す。

- **案A: 出力キャプチャ（1対1コラボ用の簡易モード）**
  - WASAPIループバック（`pyaudiowpatch` 等）でDiscord出力を第2ストリームとして取り込み、
    「自分=マイク / 相手=Discord出力」の2ボックス字幕
  - 運用: Discordの出力デバイスを専用デバイスに分けてもらい、それをループバック録音
  - 限界: ゲスト2人以上は1本のミックスに混ざるため区別不可（話者分離なしでは原理的に無理）
  - コスト: ASRが2本同時＝CPU負荷ほぼ倍（最低スペック引き上げ）。
    Discord音声は圧縮＋ノイズ抑制済みでマイク直より精度低下
- **案B: 各自がMojicastを起動し、確定字幕テキストを共有（本命）**
  - 各自のクリーンなマイク音声でローカルASR → 精度最高・話者帰属完璧・負荷分散。
    共有するのはテキストのみで帯域ほぼゼロ
  - リレー: Cloudflare Workers + Durable Objects の小さなWebSocketリレー
    （部屋コード方式・無料枠で十分。数十行規模）
  - 素朴な代替: Cloudflare Tunnel / Tailscale でホストのAPIへゲストが直接POST
    （サーバー運用ゼロ・ホストのOBSに全員分表示）
  - R2は不向き（オブジェクトストレージ＝リアルタイムpub/subに合わずポーリング前提になる）
  - 受信側の改修は小さい: リモート字幕を話者ごとにプリセット/ボックス割当して
    既存パイプラインへ流すだけ（HTTP API / SSE `/events` は既存）
- 実装順: A を簡易モード → B をコラボモード、の段階でも B 直行でも成立する
