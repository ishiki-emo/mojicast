## 1. 単語セットのプロファイル化 ＋ data/ フォルダ再編
**規模: 中（≈1日） / v0.3.0 の幹候補**

配信の性質で単語セットを切り替える（雑談用 / ゲーム実況用 / 歌枠用 …）。

- 設計: 「共通」＋ `profiles/<名前>/` の**合成**（上書きでなく加算・衝突はプロファイル優先）
  - 名前・チャンネル名は「共通」に置けば全プロファイルで有効（重複登録の回避）
- コックピットにセレクタ（`config.word_profile`）、単語スタジオに共通/プロファイル切替
- **同時に実施**: ユーザーデータを `data/` フォルダへ集約
  - `config.json`・単語系・presets/boxes → `data/`、profiles/ も `data/` 配下
  - ルートが整理され「バックアップ = dataフォルダのコピー」になる
  - **初回起動時にルートの旧ファイルを data/ へ自動移行**（既存ユーザー対策・必須）
  - `logs/` と `models/` は現行のまま
- プロファイルがそのまま mojipack（#2）の共有単位になる → **実装順は プロファイル → mojipack**

## 2. カスタマイズのユーザー間配布（mojipack）
**規模: 中 / コミュニティ化の入口**

- フェーズ1: `.mojipack`（実体はJSON）のエクスポート/インポート
  - マージ方式（上書きしない・ID再生成・名前衝突は「〜 (imported)」）
- フェーズ2: リポジトリに `community-packs/` + GitHub Discussions で募集
- フェーズ3: アプリ内ギャラリー（GitHub上のインデックス取得・急がない）
- **前提の修正**: コックピットのプリセット一覧が名前を innerHTML 描画しており、
  悪意あるパック名でHTML注入が可能。**インポート実装前にエスケープ必須**
- フォントはライセンス上同梱不可（フォント名参照＋フォールバックで対応済みの挙動を維持）

## 3. 配布ファイル数の削減（現状 onedir で約15,000ファイル）
**規模: 中〜大 / 二段構え**

- 短期: **Inno Setup インストーラ化**（≈1日）
  - 配布物が setup.exe 1個になり、展開の遅さ・MOTW・フォルダの威圧感が構造的に解消
- 長期: **推論ランタイム載せ替え**（数日＋回帰検証）
  - 句読点BERT → ONNX Runtime / FuguMT(Marian) → CTranslate2 で torch+transformers を排除
  - 約0.85GB/15,000ファイル → 約250MB/数百ファイル級へ
  - **PoC実測済み（bench/ 参照・2026-07-11）**: 移行リスクは低いと確認
    - 翻訳 CT2 fp32: 訳文12/12完全一致・約2倍速・ロード6.8s→0.3s
    - 翻訳 CT2 int8: 約5倍速(166→29ms)・60MB・差分は同格の言い換えのみ
    - 句読点 ONNX: 判定8/8完全一致・約2倍速・ロード4.9s→0.7s
  - 本移行時は実際の配信ログで回帰diffを実施すること
- onefile化は非推奨（この規模は起動毎のtemp展開が遅すぎる）

## 4. 別ソフト「話題レーダー」（仮称）
**規模: PoC半日 / 本体2〜3日 / パッケージ肥大回避のため別アプリ**

小型ローカルLLMで「今の話題」を単語抽出し、OBSにふわっと浮かべるコンパニオンアプリ。

- Mojicast側は**改修ゼロ**: SSE `http://localhost:8765/events` で確定字幕を購読できる
- 構成: 直近3〜5分のローリングウィンドウ → 30〜60秒毎にLLMで話題語3つ →
  自前ポートでOBS用オーバーレイ配信
- ランタイム: llama.cpp / GGUF（torch不要・数十MB）＋ Q4量子化モデル（初回DL方式）
- **モデル本命: Qwen3-1.7B**（CPU実試験で実用性確認済み・2026-07-12）
- CPU/GPU切替: `n_gpu_layers`（0=CPU / -1=全GPU）。バックエンドは **Vulkan推奨**
  （NVIDIA/AMD/Intel対応・同梱サイズ小。CUDAは数百MB膨らむ）
- **既定はCPU**（ゲーム配信中のGPUはゲーム+NVENCで激務）。
  「GPUを使う」トグル既定OFF＋GPU初期化失敗時はCPUフォールバック。
  雑談配信・2PC構成ではGPUが快適
- CPU配慮: 低スレッド・低優先度のバースト処理でASRを邪魔しない
- 発展: 話題語を `/api/effects` に流して字幕内を自動ハイライト（API既存）

## 5. OBS用ポートのUI編集
**規模: 小（1〜2h）**

- 裏側は対応済み: `config.json` の `"port"` は起動時に反映される（手編集なら今でも変更可）
- コックピットのOBSグループにポート欄（1024〜65535検証・次回起動時反映）
- **重要**: ポート衝突時、現状はコンソールprintで終了＝windowed exeでは無言死。
  MessageBox 表示に変える
- 変更後にOBS側ブラウザソースのURL更新が必要な旨の案内も表示
