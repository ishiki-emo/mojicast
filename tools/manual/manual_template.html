1はじめに・起動方法

Mojicast は、マイクの音声をリアルタイムに文字起こしして配信画面に字幕として表示するアプリです。 音声認識（日本語＋多言語）・句読点付け・翻訳（英・中・日・韓）・コラボ相手の字幕化まで、 すべてお使いのPCの中だけで動きます。 音声がネットに送られることはありません。

1対1コラボ字幕のデモ — ▶ 30秒デモ動画（YouTube） ／ テスト協力: 絵咲まくらさん
初回起動 初回だけ「WindowsによってPCが保護されました」(SmartScreen)が出ることがあります。 未署名アプリのためで、「詳細情報」→「実行」で起動できます。
ダウンロード由来のブロック(MOTW)はアプリが起動時に自動解除するので、手動の解除操作は不要です。
また、OSの言語やCPUに合わせたおすすめ設定の提案が出ることがあります （低スペック機の軽量構成、中国語/英語OSでの多言語認識など）。適用するかはあなた次第で、 あとからいつでも変更できます。

起動手順

1
このフォルダを丸ごとローカルディスクにコピーする（Zip 内や共有ドライブから直接起動しない）
2
Mojicast.exe をダブルクリック
3
コックピット画面が開いたら、案内の3ステップどおりに進める
初回起動時は「はじめの3ステップ」の案内が出ます
初回のみ 初回の ▶開始 時に認識モデル（約1.2GB）を自動ダウンロードし、読み込みに少し時間がかかります。 2回目以降はダウンロード無し・完全オフラインで動きます。

  • 使ってみて重いと感じたら →「困ったときは」（9章）。ワンクリックで軽くできます

CPU世代ごとの詳しい適性表・実測データ・世代の見方は、同梱の テクニカルガイド.html へ。

🧠 AIモデル（認識モデルと翻訳先）

右カラムの「AIモデル」→「✎ 設定」で開く設定窓で選びます （実際に使われる翻訳モデルと方向も窓の中に表示されます）。

項目説明
認識モデル日本語特化（既定・高精度・単語の認識誘導対応）/ 多言語（中・英・日・韓・広東語）
認識する言語多言語モデル選択時のみ。自動判定か言語固定（固定のほうが判定ミスがない）
翻訳先翻訳チップの言語（英語 / 中国語・簡体字 / 日本語 / 韓国語〔試験的〕）
認識モデルの使い分け 通常は「日本語特化」のままでOK。「多言語」の出番は ①中国語・英語など日本語以外の配信 ②低スペック機で標準モデルが重すぎるときの救済、の2つ。 多言語モデルでは「認識させる単語」の認識誘導が効かないため、固有名詞の誤変換は 読み欄への登録（置換）で直します。日本語の精度は日本語特化のほうが上です。 初回選択時にモデル（約240MB）を自動DLします。 モデルの中身や内部動作を知りたい方は テクニカルガイド.html へ。

チップ説明

ログ保存配信の文字起こしを logs\ にテキスト保存（詳細は8章）
漢数字→数字漢数字を算用数字へ変換（三十五→35 など）。「一緒」「十分(じゅうぶん)」等は変換しない
単語エフェクト強調単語の光り・アニメ・パーティクル表示をまとめてオン/オフ。オフでも単語の認識誘導・置換・伏せ字はそのまま有効
高精度認識精度がわずかに上がるが数倍重くなる。通常はオフ推奨

右カラム

  • 文字スタイル … 字幕の見た目プリセットの切替。✎編集でスタジオへ
  • レイアウト … 字幕の位置・大きさ・流れ方の切替（ミニ図は画面内での位置）
  • 単語プロファイル … 配信の種類ごとの単語セット切替（雑談用・ゲーム用など）
  • AIモデル … 認識モデル（日本語特化/多言語）と翻訳先の設定窓を開く
  • 1対1コラボ … コラボ相手の字幕化（7章）
  • OBS 連携 … OBSに貼るURLのコピーとポート変更

4OBS に字幕を出す

1
OBS で「ソース追加」→「ブラウザ」を選ぶ
2
URL に http://localhost:8765 を入力（コックピット右下の「コピー」ボタンでも取得可）
3
幅・高さを配信解像度と同じにする（例: 1920 × 1080）

アプリ側で ▶開始 すると、喋った内容が OBS 側にも表示されます。スタイルやレイアウトの変更も即座に反映されます。

表示の流れ 話している間は薄い文字で途中経過が出て、ひと呼吸おくと確定文字に変わりエフェクトが発動します。 長く喋り続けると薄文字の頭に「…」が付くことがあります（表示の省略で、確定文には全文が入ります）。

5スタジオ（見た目のカスタマイズと単語登録）

ヘッダーの「🛠 スタジオ」または各項目の「✎ 編集」から開きます。タブは4つです。

🎨 文字スタイル

  • フォント・サイズ・色・フチ・グロー・登場アニメなどを調整。下のプレビューで確認できます
  • 「複製」で自分用プリセットを作るのがおすすめ。保存でOBSへ即反映
  • コラボ用には小さめで読みやすい「コラボ（小さめ）」プリセットが最初から入っています

📐 レイアウト

  • 字幕の表示位置・サイズ・背景・行数などを定義
  • 「流れる字幕」のほか、縦書き・画面に文字が散る「リリック」モードもあります
  • コラボ用に画面下段を左右で分け合う「左下ハーフ」「右下ハーフ」が入っています
  • 各項目の意味や読みやすくするコツは、同梱の スタイルガイド.html に実例つきでまとめています

📖 単語（誤認識対策・エフェクト・伏せ字・英訳固定）

タブ役割
認識させる単語名前・チャンネル名・ゲームタイトルなど、よく誤認識される固有名詞を登録すると認識が改善。 「読み」欄にはひらがなの読みのほか、実際に出てしまう誤変換の形も入れられます （「／」区切りで複数並記OK・同じ表記で行を分けても同じ扱い）。変更は次回 ▶開始 時に反映
強調する単語特定の単語が出たとき色・サイズ・アニメ・パーティクルで装飾。「ありがとう」など定番は登録済み。保存で即反映。 全カタログと演出のコツは同梱の エフェクトガイド.html へ
伏せ字にする単語字幕に出したくない単語を「○○○」に置き換える
英訳を固定する単語人名などの英訳を固定（例: 星野ひかり → Hikari Hoshino）。次回開始時に反映

上部の「編集する単語セット」で単語プロファイル（雑談用・ゲーム用など）を作れます。 プロファイルは「共通」への加算なので、いつも使う単語は共通に入れておけば全プロファイルで有効です。

🔗 共有（mojipack）

「📤 エクスポート」で文字スタイルやレイアウトを .mojipack ファイルに書き出して配布できます。 もらったファイルは「📥 インポート」で取り込み。既存の定義は上書きされず追加されます。

6翻訳の併記（英訳・中国語訳）

字幕ログ上の翻訳チップをオンにすると、確定した字幕の下に翻訳が小さく併記されます。 海外リスナー向けの配信にどうぞ。翻訳先は右カラムの「AIモデル → ✎ 設定」で切り替えられます （チップ名も「英訳」⇔「中国語訳」に変わります）。

翻訳先モデル補足
英語（既定）FuguMT 人名などはスタジオの「英訳を固定する単語」で訳を固定できます
中国語（簡体字）M2M-100 選択後の初回 ▶開始 時にモデルを自動DL（約470MB）。「英訳を固定する単語」は適用されません
日本語M2M-100 多言語認識（中・英・韓）と組み合わせて、海外の配信を日本語字幕にする方向。認識言語は固定推奨
韓国語M2M-100 試験的。翻訳が不安定なことがあります

広東語は認識のみ対応です（翻訳モデル側が非対応）。認識言語と同じ翻訳先を選ぶと翻訳はスキップされます。

  • 翻訳もローカルで動きます（ネット不要・追加料金なし）
  • 日本語の字幕が先に出て、翻訳がひと呼吸おいて追いつきます
  • オンにして最初の ▶開始 時だけ「翻訳モデルをロード中...」と数秒待ちが入ります
  • 翻訳の見た目（サイズ・色・フチ）は文字スタイルの「英訳」設定が使われます（中国語訳も同じ）
配信用語は組み込みで対応 「配信→stream」「スパチャ→Super Chat」「歌枠→singing stream」（英訳）、 「配信→直播」「コメ欄→评论区」（中国語訳）など、 誤訳されがちな配信用語はアプリに組み込みで正しく訳されます。
品質について 軽量な翻訳モデルのため、辞書に無い固有名詞はうまく訳せないことがあります。 気になった誤訳があればぜひフィードバックしてください。

71対1コラボ字幕

Discord等で通話しながらのコラボ配信で、自分と相手の両方の字幕を別々のスタイル・位置で表示できます。 相手の声は「通話アプリの音」から直接取り込むので、相手側の準備は一切不要です （相手がスマホ・ゲーム機でもOK）。 ▶ 実際に動く様子は30秒デモ動画（YouTube）をどうぞ。

できること制約
相手はアプリ導入・設定不要
相手の字幕に専用のスタイル・位置を割当
字幕ログに話者名チップ表示		 1対1限定（3人以上は相手全員の声が1つにまとまる）
相手の音は通話品質のため、認識精度は自分よりやや低め
Windows 10 2004 以降が必要

設定手順

1
右カラムの「1対1コラボ」で「コラボを有効化」にチェック → 設定窓が自動で開く
2
Discord等の通話アプリを起動してから ↻ を押し、 「相手の声が鳴るアプリ」で Discord.exe などを選ぶ（🔊 は今音が鳴っているアプリ）
3
自分と相手の表示名を決める（字幕ログのチップに使われます）
4
コックピットに戻り、「文字スタイル」「レイアウト」の「相手」の行で相手の見た目を選ぶ。 おすすめは 自分=右下ハーフ / 相手=左下ハーフ（立ち絵が左右に並ぶ配置向け）
5
ヘッドホンを着けて ▶開始
コラボ設定窓（取り込むアプリと表示名）
コラボ中は「自分」「相手」の2行になる
コラボ中のコックピット。字幕ログに話者名チップが付きます
ヘッドホン必須 スピーカーで聞くと、相手の声をマイクが拾って二重に字幕化されます。コラボ中は必ずヘッドホン・イヤホンで。
負荷について コラボ中は認識が2人分になるため、PCによっては重くなります。 重い場合は認識モデルを「多言語」にするのが効果的（2人分でも負荷がほぼ増えません。精度は少し下がります）。 詳しい実測データは テクニカルガイド.html へ。 重い場合は「高精度」チップをオフ、英訳をオフ、「一区切りの最長」を8秒程度に。

8フォルダとアップデート

ファイル / フォルダ内容
Mojicast.exeアプリ本体
_internal\実行に必要なファイル（触らない）

文字起こしログ（logs フォルダ）

字幕ログ上の「ログ保存」チップがオンのとき（既定オン）、配信の文字起こしが テキストで残ります。

  • 保存先は logs\日付\ フォルダ。▶開始のたびに1ファイルできます （例: logs\2026-07-20\2026-07-20_161916.log）
  • 各行は [発言時刻] 本文 の形式。コラボ中は [話者名] も付くので、 誰の発言かあとから分かります
  • 確定した字幕だけが記録されます（薄文字の途中経過は残りません）。 伏せ字にした単語は伏せ字のまま記録されます
  • 書き込みは発言ごとなので、アプリが落ちてもそれまでの分は残ります
  • 自動では削除されません。溜まってきたら古い日付フォルダをそのまま削除してOKです
活用アイデア 配信のアーカイブ字幕づくり・切り抜きの台本探し・「あの話いつしたっけ？」の検索 （テキストなのでフォルダごと全文検索できます）など。

アップデート方法

1
Mojicast を終了する
2
旧フォルダ内の _internal フォルダを削除する
3
新しいZipを同じフォルダに上書き展開する

設定・単語帳・スタイル・AIモデル（models）はそのまま引き継がれます。 新しいZipにはデータファイルが含まれないため、あなたのカスタマイズは上書きされません。 アップデートで追加された新しい既定スタイルは、初回起動時に自動で一覧へ追加されます。

アンインストールはフォルダごと削除するだけです。レジストリ等は使いません。

9困ったときは

症状対処
動作が重い① 認識モデルを「多言語」に（負荷が約1/3・精度は少し下がる。右カラムの「AIモデル」から） ②「高精度」チップをオフ ③ 使わないなら翻訳をオフ ④ 詳細設定の「一区切りの最長」を8秒程度に。 CPU使用率が上がるのは「誰かが喋っている間」だけです。詳しくは テクニカルガイド.html
画面が真っ白「Microsoft Edge WebView2 ランタイム」をインストール（Web検索で公式Microsoftサイトから）。Windows 11 では通常発生しません
「ポート ○○ は使用中」アプリが既に起動しています。タスクトレイやタスクマネージャーで確認して二重起動を解消。 別アプリとの衝突ならコックピットの「OBS連携 → ポート」で変更（次回起動時反映・OBS側のURLも更新）
マイクを拾わないWindows 設定 → プライバシー → マイクの許可を確認。他アプリの占有も確認。マイク選択を「既定のマイク」に戻すのも有効
