1はじめに・起動方法

Mojicast は、マイクの音声をリアルタイムに文字起こしして配信画面に字幕として表示するアプリです。 音声認識（日本語＋多言語）・句読点付け・翻訳（英・中・日・韓・インドネシア）・コラボ相手の字幕化まで、 すべてお使いのPCの中だけで動きます。 音声がネットに送られることはありません。画面はライト基調です （ヘッダーの 🌙/☀ でダークにも切替可・字幕そのものの色には影響しません）。

1対1コラボ字幕のデモ — ▶ 30秒デモ動画（YouTube） ／ テスト協力: 絵咲まくらさん
初回起動 初回だけ「WindowsによってPCが保護されました」(SmartScreen)が出ることがあります。 未署名アプリのためで、「詳細情報」→「実行」で起動できます。
ダウンロード由来のブロック(MOTW)はアプリが起動時に自動解除するので、手動の解除操作は不要です。
また、OSの言語やCPUに合わせたおすすめ設定の提案が出ることがあります （低スペック機の軽量構成、中国語/英語OSでの多言語認識など）。適用するかはあなた次第で、 あとからいつでも変更できます。

起動手順

1
このフォルダを丸ごとローカルディスクにコピーする（Zip 内や共有ドライブから直接起動しない）
2
Mojicast.exe をダブルクリック
3
コックピット画面が開いたら、中央の「はじめの3ステップ」どおりに進める
起動直後のコックピット。中央に「はじめの3ステップ」が出ます（ライト基調）
初回のみ 初回の ▶開始 時に認識モデル（約1.2GB）を自動ダウンロードし、読み込みに少し時間がかかります。 2回目以降はダウンロード無し・完全オフラインで動きます。
⚠ ご利用にあたって 認識・翻訳・伏せ字は完璧ではなく、不適切な表示が出ることがあります（大事な場面はご自身でも確認を）。
1対1コラボは相手の声を字幕化・記録するので、相手の同意を得てご利用ください。
本アプリは無保証（自己責任）です。詳しい免責は同梱の README を参照してください。

🎙 声の聞き取り

項目説明
使うマイク音声入力デバイスの選択
認識モデル日本語特化（おすすめ）…高精度・単語の認識誘導対応 ／ 多言語…中・英・日・韓・広東語
認識する言語多言語モデル選択時のみ。自動判定か言語固定（固定のほうが判定ミスがない）
字幕の区切り話し終わりの区切り（この長さ黙ると1行が確定）／一区切りの最長（強制確定の秒数・重いPCでは短め推奨）／途中経過の更新間隔（薄文字の更新頻度）
認識の補助登録した単語を認識に使う／登録単語の出やすさ（効かない→上げる/誤爆→下げる）／漢数字を数字にする（三十五→35。「一緒」等は変換しない）／高精度モード（精度がわずかに上がるが数倍重い・通常オフ推奨）
認識モデルの使い分け 通常は「日本語特化」のままでOK。「多言語」の出番は ①中国語・英語など日本語以外の配信 ②動作を軽くしたいとき、の2つ。 多言語モデルでは「認識させる単語」の認識誘導が効かないため、固有名詞の誤変換は 読み欄への登録（置換）で直します。日本語の精度は日本語特化のほうが上です。 初回選択時にモデル（約240MB）を自動DL。モデルの中身は テクニカルガイド.html へ。
反映タイミング マイク・認識まわりは次回 ▶開始 時に反映されます。認識中に変更すると黄色い警告バーが出るので、 コックピットの「今すぐ反映（再起動）」ボタンを押せばその場で反映できます（停止→開始を自動でやるだけ）。

5OBS に字幕を出す

1
OBS で「ソース追加」→「ブラウザ」を選ぶ
2
URL に http://localhost:8765 を入力（コックピット右カラムの「OBSに表示」からコピー可）
3
幅・高さを配信解像度と同じにする（例: 1920 × 1080）

アプリ側で ▶開始 すると、喋った内容が OBS 側にも表示されます。スタイルやレイアウトの変更も即座に反映されます。 貼る前の見え方はコックピットの配信プレビューで確認できます。

表示の流れ 話している間は薄い文字で途中経過が出て、ひと呼吸おくと確定文字に変わりエフェクトが発動します。 長く喋り続けると薄文字の頭に「…」が付くことがあります（表示の省略で、確定文には全文が入ります）。

6スタジオ（字幕の見た目と言葉）

コックピットの 🎨 字幕スタジオ、または各項目の カスタマイズ/設定 から開きます。 左メニュー「つくるもの」で 字幕 / 翻訳字幕 / ワード演出 / 保存・共有 に分かれます。 右上の「編集する配信セット」は単語系にだけ効くプロファイル切替です。

💬 字幕（見た目・位置・認識単語）

  • 文字スタイル … フォント・色・フチ・グロー・登場アニメ等。「⧉ 新しいデザイン（複製）」で自分用プリセットを作るのがおすすめ。保存でOBSへ即反映
  • レイアウト … 表示位置・背景・行数など。表示のしかたで 通常（積み上げ・スクロール）／縦書き（右から左へ）／リリックビデオ風字幕 を選べます
  • コラボ用に「コラボ（小さめ）」スタイルと「左下ハーフ」「右下ハーフ」レイアウトが最初から入っています
  • 各項目の意味や読みやすくするコツは、同梱の スタイルガイド.html に実例つきでまとめています

7翻訳の併記

コックピット右カラムの「英訳（翻訳）」トグルをオンにすると、確定した字幕の下に翻訳が小さく併記されます。 海外リスナー向けの配信にどうぞ。翻訳先は「アプリ設定 → 翻訳」で切り替えられます （コックピットのトグル名も翻訳先に合わせて変わります）。

翻訳先モデル補足
英語（既定）FuguMT 人名などはスタジオの「英訳を固定する単語」で訳を固定できます
中国語（簡体字・中国大陸）M2M-100 選択後の初回 ▶開始 時にモデルを自動DL（約470MB）。「英訳を固定する単語」は適用されません
中国語（繁体字・台湾）M2M-100 ＋ OpenCC 翻訳後に台湾正体字と台湾で一般的な語彙へ整えます
中国語（繁体字・香港）M2M-100 ＋ OpenCC 翻訳後に香港繁体字へ整えます
インドネシア語M2M-100 日本語から直接翻訳します
日本語M2M-100 多言語認識（中・英・韓）と組み合わせて、海外の配信を日本語字幕にする方向。認識言語は固定推奨
韓国語M2M-100 試験的。翻訳が不安定なことがあります

広東語は繁体字の種類ではなく別の言語です。現在は音声認識のみ対応し、翻訳先としては未対応です。認識言語と同じ翻訳先を選ぶと翻訳はスキップされます。

  • 翻訳もローカルで動きます（ネット不要・追加料金なし）
  • 日本語の字幕が先に出て、翻訳がひと呼吸おいて追いつきます
  • オンにして最初の ▶開始 時だけ「翻訳モデルをロード中...」と数秒待ちが入ります
  • 翻訳字幕の見た目（サイズ・色・フチ）はスタジオの「翻訳字幕の見た目」で本文と独立して調整できます
配信用語は組み込みで対応 「配信→stream」「スパチャ→Super Chat」「歌枠→singing stream」（英訳）、 「配信→直播」「コメ欄→评论区」（中国語訳）など、 誤訳されがちな配信用語はアプリに組み込みで正しく訳されます。
訳の精度について 翻訳はPC内で完結する軽量なローカルモデルで動くため、精度はクラウド翻訳ほどではなく 「意味が伝わればOK」くらいのそれなりとお考えください。長い一文や固有名詞・スラングは崩れやすいです。 人名などは「英訳を固定する単語」で訳を固定でき、配信でよく使う語は組み込み辞書で補正しています。 気になった誤訳があればぜひフィードバックしてください。

81対1コラボ字幕

Discord等で通話しながらのコラボ配信で、自分と相手の両方の字幕を別々のスタイル・位置で表示できます。 相手の声は「通話アプリの音」から直接取り込むので、相手側の準備は一切不要です （相手がスマホ・ゲーム機でもOK）。 ▶ 実際に動く様子は30秒デモ動画（YouTube）をどうぞ。

できること制約
相手はアプリ導入・設定不要
相手の字幕に専用のスタイル・位置を割当
字幕ログに話者名チップ表示		 1対1限定（3人以上は相手全員の声が1つにまとまる）
相手の音は通話品質のため、認識精度は自分よりやや低め
Windows 10 2004 以降が必要

設定手順

1
コックピット右カラムの「1対1コラボ」トグルをオン → アプリ設定の「コラボ音声」が開く
2
Discord等の通話アプリを起動してから ↻ を押し、 「相手の声が鳴るアプリ」で Discord.exe などを選ぶ（🔊 は今音が鳴っているアプリ）
3
自分と相手の表示名を決める（字幕ログのチップに使われます）
4
コックピットに戻ると「今日の字幕」が自分/相手の2段になるので、相手の字幕デザイン・字幕の位置を選ぶ。 おすすめは 自分=右下ハーフ / 相手=左下ハーフ（立ち絵が左右に並ぶ配置向け）
5
ヘッドホンを着けて ▶開始
アプリ設定「コラボ音声」（取り込むアプリと表示名）
コラボ中は「今日の字幕」が自分/相手の2段になります
ヘッドホン必須 スピーカーで聞くと、相手の声をマイクが拾って二重に字幕化されます。コラボ中は必ずヘッドホン・イヤホンで。
負荷について コラボ中は認識が2人分になるため、PCによっては重くなります。 重い場合は認識モデルを「多言語」にするのが効果的（2人分でも負荷がほぼ増えません。精度は少し下がります）。 詳しい実測データは テクニカルガイド.html へ。 ほかに「高精度モード」をオフ、英訳をオフ、「一区切りの最長」を8秒程度にするのも有効です。

文字起こしログ（logs フォルダ）

アプリ設定の「その他 → 字幕ログを保存する」がオンのとき（既定オン）、配信の文字起こしが テキストで残ります。

  • 保存先は logs\日付\ フォルダ。▶開始のたびに1ファイルできます （例: logs\2026-07-20\2026-07-20_161916.log）
  • 各行は [発言時刻] 本文 の形式。コラボ中は [話者名] も付くので、 誰の発言かあとから分かります
  • 確定した字幕だけが記録されます（薄文字の途中経過は残りません）。 伏せ字にした単語は伏せ字のまま記録されます
  • 書き込みは発言ごとなので、アプリが落ちてもそれまでの分は残ります
  • 自動では削除されません。溜まってきたら古い日付フォルダをそのまま削除してOKです
活用アイデア 配信のアーカイブ字幕づくり・切り抜きの台本探し・「あの話いつしたっけ？」の検索 （テキストなのでフォルダごと全文検索できます）など。

アップデート方法

1
Mojicast を終了する
2
旧フォルダ内の _internal フォルダを削除する
3
新しいZipを同じフォルダに上書き展開する

設定・単語帳・スタイル・AIモデル（models）はそのまま引き継がれます。 新しいZipにはデータファイルが含まれないため、あなたのカスタマイズは上書きされません。 アップデートで追加された新しい既定スタイルは、初回起動時に自動で一覧へ追加されます。

アンインストールはフォルダごと削除するだけです。レジストリ等は使いません。

10困ったときは

症状対処
動作が重い① 認識モデルを「多言語」に（負荷が約1/3・精度は少し下がる。アプリ設定→声の聞き取り→認識モデル） ②「高精度モード」をオフ ③ 使わないなら翻訳をオフ ④「一区切りの最長」を8秒程度に。 CPU使用率が上がるのは「誰かが喋っている間」だけです。詳しくは テクニカルガイド.html
画面が真っ白「Microsoft Edge WebView2 ランタイム」をインストール（Web検索で公式Microsoftサイトから）。Windows 11 では通常発生しません
「ポート ○○ は使用中」アプリが既に起動しています。タスクトレイやタスクマネージャーで確認して二重起動を解消。 別アプリとの衝突ならアプリ設定の「OBS接続 → 接続ポート」で変更（次回起動時反映・OBS側のURLも更新）
マイクを拾わないWindows 設定 → プライバシー → マイクの許可を確認。他アプリの占有も確認。マイク選択を「既定のマイク」に戻すのも有効
認識が変・遅い詳細設定の「無音で文を区切る」「途中経過の更新間隔」を調整。固有名詞の誤認識はスタジオの「認識させる単語」で改善
コラボで相手の字幕が出ない通話アプリを起動してからコラボ設定の ↻ →選び直し。相手が喋ったときに出ます。 アプリを選び直したら、いったん ■停止 → ▶開始
コラボで字幕が二重に出るスピーカーの音をマイクが拾っています。ヘッドホンを使ってください
OBS に何も出ないアプリが起動しているか、URL が http://localhost:8765 か、▶開始 しているかを確認。ブラウザソースを右クリック→「再読み込み」も有効
多言語モデルで固有名詞が直らない認識誘導が効かないモデルのため、「認識させる単語」の読み欄に実際に出た誤変換の形を登録します（「／」区切りで複数OK）
翻訳が出ない認識言語と翻訳先が同じだとスキップされます。アプリ設定→翻訳の「使用モデル」表示を確認
「句読点の読み込みに失敗」
「翻訳の読み込みに失敗」と出る		字幕本体はそのまま動きます。原因が logs\load_error.log に記録されるので、 続く場合はこのファイルを添えて報告してください
起動時に .NET / clr / Python.Runtime のエラー通常は起動時に自動解除されますが、まれに残ることがあります。フォルダ内の ブロック解除.bat を一度実行してください
起動しない・すぐ落ちるフォルダをローカルディスクに置いているか確認（Zip内・ネットワークドライブは不可）。ウイルス対策ソフトの誤検知の場合は除外設定へ

上級者向け: 認識中にブラウザで http://localhost:8765/api/perf を開くと、 認識処理の回数と平均所要時間(ms)を確認できます。不具合報告の際に添えていただけると原因特定が早くなります。
