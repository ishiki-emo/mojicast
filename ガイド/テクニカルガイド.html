1. CPUと性能

適性の目安

Mojicastの負荷の本体は音声認識のAI推論（int8）で、CPUの世代によって速さが数倍違います。 「コア数」より「世代（対応命令）」が効きます。

適性 CPU の例 目安
◎ 快適 AMD Ryzen 7000 / 9000（Zen 4/5） 余裕。コラボ・翻訳を全部ONでも軽い
○ 実用 Intel 12〜14世代 / Core Ultra ／ AVX-512対応の Intel 10〜11世代※ 通常配信は問題なし。コラボで2人とも喋りっぱなしだと負荷高め
△ 動くが重め Intel 8〜10世代（AVX-512非対応モデル） / Ryzen 2000〜5000 ソロ配信なら実用。軽量化設定を推奨
✕ 非推奨 AVX2非対応の古いCPU・N100等の省電力CPU 標準モデルでは連続発話に追いつかない。多言語モデルなら条件付きで動作（下の実測）

※ 11世代（デスクトップ Rocket Lake / ノート Tiger Lake）と10世代ノート（Ice Lake）は int8 推論専用命令の AVX-512 VNNI に対応しており、見た目の世代より速く動きます （10世代デスクトップ Comet Lake は非対応なので △）。

この命令を使えるCPU（Zen 4以降・上記のIntel 10〜11世代）は認識処理が数倍速く走ります。 Intel 12世代以降ではAVX-512が無効化され、256bit版の AVX-VNNI に置き換わりました（それでも○）。 CPU使用率は「誰かが喋っている間」だけ上がり、無音の間はほぼゼロです。

実測例（テスター報告・ソロ配信時）: - Intel Core i7-13700HX（16コア・○の帯）— 認識中のCPU使用率 33〜40%（翻訳OFF・通常精度）。 翻訳ON＋高精度の全部盛りでも 37〜45%。多言語モデルなら翻訳ON込みで 13〜20% - Intel N100（省電力4コア・✕の帯）— 標準モデル（k2）は CPU 90%超に張り付き、連続発話で 字幕の遅れが蓄積して実用不可。多言語モデルなら CPU 80〜90% で概ね追従 （長い文の直後は次の文が遅れるが回復する）。メモリは約1.3GB（ボトルネックはCPU馬力のみ） - コラボ（認識2人分）でも、多言語モデルは負荷がほぼ増えないことを実測確認

世代の見方（自分のCPUがどれか分からないとき）

自分のCPU名は「設定 → システム → バージョン情報」で見られます。

CPU名の例 世代の読み方
Intel Core i9-11900 / i5-10400 ハイフンの後ろ、末尾3桁を除いた数字が世代（11900→11世代、10400→10世代、8700→8世代）
Intel Core i7-1165G7（ノート） 同じく先頭2桁が世代（1165G7→11世代。末尾G7はIce Lake/Tiger Lake系＝AVX-512対応の目印）
Intel Core Ultra 7 155H 「Core Ultra」と付けば新世代（12世代以降相当）で ○
AMD Ryzen 7 7700X（デスクトップ） 先頭1桁が数字世代。7000/9000番台なら ◎
AMD Ryzen 7 7840HS（ノート） ノート用は中身が混在。十の位が4以上（7640/7840/8840等）なら Zen 4 相当で ◎

初回起動時の「おすすめ設定」は、この判定をアプリが自動で行って提案しています （確信のない型番では何も提案しません）。

重いときの選択肢（効果順）

  1. 認識モデルを「多言語」に（負荷が約1/3。精度はわずかに下がる）
  2. 「高精度」チップをオフ（既定でオフ）
  3. 翻訳チップをオフ
  4. 詳細設定の「一区切りの最長」を8秒に
  5. コラボ時: 上記1が特に有効（2人分でも負荷がほぼ増えない）

2. 搭載AIモデル

すべてローカル動作・初回選択時に自動ダウンロードされます。ライセンスの詳細は CREDITS を参照。

役割 モデル DLサイズ 備考
日本語認識（標準） ReazonSpeech k2 v2 約740MB 最高精度。「認識させる単語」の認識誘導（コンテキストバイアス）対応
多言語認識 SenseVoice small 約240MB 中・英・日・韓・広東語。句読点・数字正規化を内蔵。非自己回帰型で軽い
発話区間検出 Silero VAD 同梱 言語非依存
句読点 日本語BERT 約360MB 標準モデル使用時のみ（多言語モデルは句読点内蔵のため未使用）
日→英翻訳 FuguMT 約120MB 配信用語の組み込み辞書つき
多方向翻訳 M2M-100 418M 約470MB 中・日・韓方向。100言語相互翻訳モデル

多言語モデル選択時の内部動作

  • 句読点BERT・漢数字変換は自動でスキップ（モデルが句読点と数字正規化を内蔵しているため）。 未使用のBERTはメモリからも解放されます
  • 認識結果のCJK文字間に入る余分な空白は自動除去（英語の空白は保持）
  • 「認識させる単語」の認識誘導は効きません（transducer構造専用の機能のため）。 代わりに読み欄→表記の置換が全モデル共通で働きます。読み欄は「／」区切りで 複数の誤変換形を並記でき、単語の途中に句読点が挟まっても照合されます

3. 翻訳の経路

翻訳モデルは「認識している言語 × 翻訳先」から自動選択されます （AIモデル設定窓に「使用モデル: 〜」として表示）。

  • 日→英 だけは日英特化の FuguMT、それ以外の方向はすべて M2M-100
  • 認識言語と翻訳先が同じ組み合わせは翻訳をスキップ
  • 広東語認識時の翻訳は、書き起こしが中文字であることを利用して中国語扱いで翻訳
  • 韓国語訳はgreedy探索の反復暴走対策として反復ペナルティを自動適用（それでも品質は発展途上のため「試験的」表記）
  • 翻訳は認識とは別スレッドで動くため、翻訳が遅れても字幕本体は詰まりません
