# UI 再設計メモ（導線整理＋テーマ）

機能が増えてきたため、実装を進める前にUI/導線を整理する。方針を確定し、
モックで見た目まで合意した内容の記録。実装計画つき。

対話モック（実物の配色・触れる3ビュー）:
https://claude.ai/code/artifact/06b78011-b00e-446b-9230-2361d556d384

## 1. 現状の課題（実物ベース）

3窓構成: コックピット（メイン）/ 単語スタジオ（別窓）/ スタイルスタジオ（別窓）。

1. **「選ぶ場所」と「作る場所」が窓をまたいで分散** — 文字スタイル・レイアウト・
   単語プロファイルは選択がコックピット、編集が別窓。ルールは一貫しているのに
   明示されておらず「編集どこ？」が毎回迷う。入口も散在
2. **用語がバラバラ** — レイアウト＝「ボックス（配置）」＝「BOXES」、
   文字スタイル＝「PRESETS」。日本語↔英語・複数呼称で同一物と気づけない
3. **コックピット左が過密** — 難しい認識パラメータ（スライダー4種）と日常トグル
   （6種）が同列。初心者が触るべきでない値が前面
4. **別窓が2つ増殖しやすい** — webviewの独立窓でAlt+Tab混雑。機能増で悪化方向
5. **コラボで右カラムが崩れる** — 案A改/案Bは「話者ごとにスタイル・ボックス割当」。
   現状は「1プリセット・1ボックスをcfgで選ぶ」前提。今整理しないと破綻する

## 2. 整理方針（確定）

1. **「選ぶ」と「作る」を役割で分ける** — コックピット＝“今使うものを選ぶ”操作ハブ、
   スタジオ＝“作る・育てる”作業場。右カラム各カードに「スタジオで編集 →」を置き往復を明快に
2. **別窓 2 → 1（スタジオ統合）** — 単語スタジオ＋スタイルスタジオを1つの STUDIO 窓に。
   上部タブで「文字スタイル／レイアウト／単語／共有」を切替
3. **用語を統一** — 「文字スタイル」「レイアウト」に一本化（英語見出し PRESETS/BOXES は廃止）
4. **基本と詳細を階層化** — 難しい認識パラメータ（無音で確定・途中更新間隔・最大発話長・
   単語ブースト・高精度・算用数字）は「⚙ 詳細設定」に格納。前面は日常トグル
   （句読点・英訳・ログ）だけ
5. **コラボを先回りして器を用意** — コックピット右に「話者（コラボ）」枠を将来枠として先置き。
   案A改・案B共通で話者→スタイル/レイアウト割当へ拡張しやすくしておく
   （[COLLAB_DESIGN.md](COLLAB_DESIGN.md) の話者付き描画基盤と接続）
6. **ライト／ダークのテーマ選択（新規）** — 下記3節

## 3. テーマ（ライト／ダーク）

- **2択でシンプルに**（ダーク／ライト。「自動（OS追従）」は入れない）
- **既定はダーク**（現行踏襲・配信者はダーク好みが多い）
  → ※v0.5系で**既定をライトに変更**（初見の親しみやすさ優先）。ダークは 🌙/☀ でいつでも選択可
- **切替の置き場所**: コックピット/スタジオの**ヘッダ右に常設アイコン**（🌙/☀）。すぐ切れる
- **対象はGUI窓のみ**（コックピット・スタジオ）。**overlay（OBS表示）は透過なので対象外**
- **字幕モニタ・配置プレビューは透過オーバーレイの見た目** — ライトでは黒ベタをやめ、
  透過を示す**市松模様**にする。字幕は実際どおり白文字＋黒フチで、どの背景でも読める
- ライト用アクセントは白地で読めるよう沈める（cyan `#0499bd` / magenta `#d4247a`）

### 実装方針
- 各HTMLのハードコード色を **CSS変数（トークン）へ寄せる**（現状 cockpit は変数化済み、
  words/style も同様に整える）
- `:root` にダーク既定、`:root[data-theme="light"]`（または html属性）でライトを上書き
- テーマ値は config に保存し、全GUI窓へ適用（`/api/config` に `theme` を追加）

## 4. 目標IA

```
コックピット（操作ハブ・選ぶ）
  ヘッダ: ロゴ / 状態LED / [表示クリア] [🛠 スタジオ] [🌙/☀] [▶開始]
  左「入力」  : マイク / 入力レベル / 単語プロファイル（→編集リンク）/
                日常トグル（句読点・英訳・ログ）/ ⚙詳細設定（畳む: 認識パラメータ）
  中央        : 字幕モニタ
  右「表示」  : 文字スタイル選択（→編集）/ レイアウト選択（→編集）/
                話者（コラボ・将来枠）/ OBS連携（URL・ポート）

スタジオ（作る・1窓）
  ヘッダ: STUDIO / プロファイル選択（単語にだけ効く）/ [💾 保存]
  タブ  : 🎨 文字スタイル / 📐 レイアウト / 📖 単語 / 🔗 共有
    - 文字スタイル : プリセット一覧＋エディタ（旧 style.html の style タブ）
    - レイアウト   : ボックス一覧＋エディタ（旧 style.html の box タブ）
    - 単語         : ホットワード/エフェクト/禁止/英訳辞書（旧 words.html の4タブ）
                     ＋プロファイル管理。※単語はプロファイル単位
    - 共有         : mojipack エクスポート/インポート（＋将来: 単語パック）
```

補足: 文字スタイル・レイアウトは全体共通（presets.json / boxes.json）、
単語はプロファイル単位。スタジオ上部の「プロファイル」は単語タブにだけ効く。

## 5. 実装計画（タスク）

| # | 内容 | 該当 | 規模 |
|---|---|---|---|
| 1 | 用語統一（レイアウト／文字スタイルへ・英語見出し廃止） | cockpit/words/style | 小 |
| 2 | コックピット左を「基本トグル＋⚙詳細設定（折りたたみ）」に再編 | cockpit.html | 小〜中 |
| 3 | 右カラム各カードに「スタジオで編集 →」導線を追加 | cockpit.html + app.py JsApi | 小 |
| 4 | 単語スタジオ＋スタイルスタジオを1窓 STUDIO に統合（タブ化） | words/style → studio.html | 中 |
| 5 | JsApi の窓管理を studio 1本に（open_words/open_style を統合） | app.py | 小 |
| 6 | テーマ: 色のCSS変数化＋`data-theme`適用＋config `theme` 保存 | 全GUI HTML + app_server.py | 中 |
| 7 | ヘッダにテーマ切替（🌙/☀）常設 | cockpit/studio | 小 |
| 8 | （コラボ実装時）右「話者」枠を実機能化（COLLAB_DESIGN #4・#5と接続） | cockpit + overlay | 別途 |

- 先にやると楽な順: 1（用語）→ 6/7（テーマ・色の変数化はどのみち触る）→ 2/3（コックピット再編）→ 4/5（統合）
- 統合（#4）が一番の山。旧2窓のHTML/JSをタブ内へ移植し、保存/プロファイルの扱いを共通化
- コラボ（#8）はUI基盤が整ってから。話者枠は先にプレースホルダとして置く
